# 如何改成真实接口（已完成）

> **当前状态：患者端已全部改为真实接口联调模式。**  
> 详细对照见 [小程序功能与后端接口对照.md](./小程序功能与后端接口对照.md)

## 你现在只需要做 3 件事

### 1. 改后端地址

`miniprogram/utils/config.ts`：

```typescript
export const API_BASE = 'http://你的IP:8080/api'
```

### 2. 启动后端

确保对照文档第五节列出的接口已实现。

### 3. 微信开发者工具

- 勾选「不校验合法域名」
- 先登录患者账号，再测挂号 / 缴费 / AI / 病历

## 代码结构（以后扩展用）

```text
miniprogram/utils/
├── config.ts          # API_BASE、开关
├── request.ts         # 统一请求
├── format.ts          # 列表解包、状态文案
├── patient.ts         # patientId、登录检查
└── api/
    ├── auth.ts
    ├── department.ts
    ├── doctor.ts
    ├── registration.ts
    ├── registration-ui.ts
    ├── payment.ts
    ├── ai.ts
    └── record.ts
```

**改一个新页面的标准步骤：**

1. 在 `api/` 下写 `fetchXxx()`  
2. 页面 `onShow` 里 `await fetchXxx()` → `setData`  
3. 在对照文档里补一行

## Mock 已关闭

```typescript
export const ENABLE_MOCK_FALLBACK = false
```

后端失败时：**不会**再提示「演示成功」，只会显示错误 or 空列表。

若本地没后端想临时演示，可改回 `true`（不推荐用于正式联调）。
