## 五、对话中解决的问题

### 5.1 编译与 UI

| 问题 | 原因 | 处理 |
|------|------|------|
| 模拟器缺 `home.js`、白屏 | TS 未编译；`es6: false` | `project.config.json` 设 `es6: true` |
| 首页布局错乱、搜索与轮播重叠 | `scroll-view` + 负 margin | 改为普通页面滚动，调整间距 |
| 快捷服务文字竖排/裁切 | 组件 flex | 首页改为 4 列 grid |
| Tab「首页」一直蓝色 | 未选中图标也是蓝色 | 重做 gray/blue 图标 |
| AI Tab 登录死循环 | Tab 页 `onShow` 强制跳登录 | Tab 用 `checkLogin({ navigate: false })`；401 不自动跳登录 |

### 5.2 联调与请求

| 问题 | 原因 | 处理 |
|------|------|------|
| 合法域名校验 | `localhost` 不在白名单 | `urlCheck: false` + 开发者工具勾选不校验域名 |
| request timeout | 模拟器对 `localhost` 不稳定 | 改用 `127.0.0.1`；`useApiHostProcess: false` |
| 登录报「响应格式异常」 | 登录返回 `{ accessToken }` 无 `code` 包装 | `request.ts` 兼容两种响应格式 |
| 登录后科室仍 403 | `needAuth: false` 不带 Token | **有 Token 就自动带上**；科室/医生去掉 `needAuth: false` |

### 5.3 范围重构（按 ENTITY 文档）

- Tab 改为 3 个；去掉 AI 导诊、CT
- API 路径对齐：`/registrations/quick`、`/payment/pending` 等
- 挂号详情增加「到院签到」
- 新增 `pages/patient/profile` 编辑资料
- 更新 `docs/小程序功能与后端接口对照.md`

## 八、重要代码约定

```typescript
// Tab 页 onShow：只判断登录态，不跳转
checkLogin({ navigate: false })

// 按钮/子页面：需要登录再跳转
checkLogin({ navigate: true })

// 401：request.ts 只清 Token + Toast，不 navigateTo 登录页
```

登录成功存储：

- `accessToken` / `refreshToken` → `wx.setStorageSync`
- 用户信息 → `getApp().setUserInfo()`
