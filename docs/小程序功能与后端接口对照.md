## 六、统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": "2026-06-11T10:00:00Z"
}
```

分页 `data` 结构：

```json
{
  "records": [],
  "pageNo": 1,
  "pageSize": 10,
  "total": 100
}
```

`utils/format.ts` 中 `unwrapList()` 已兼容「直接数组」和「分页 records」两种格式。

## 八、常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| 列表为空 | 后端无数据或接口路径不一致 | 查 Network 响应 |
| 网络异常 | 后端未启动 / 地址错误 | 检查 `API_BASE` |
| 401 跳登录 | Token 失效或未登录 | 重新登录 |
| 字段显示空白 | 后端字段名与前端映射不一致 | 在 `api/*.ts` 或页面 map 函数中适配 |
| 检查/检验列表空 | 后端可能仅有详情接口无列表 | 需后端补充分页列表或改查申请单接口 |

## 九、改接口地址 / 字段适配

**改地址：** 只改 `miniprogram/utils/config.ts` 的 `API_BASE`。

**改字段映射：** 在对应 `utils/api/*.ts` 或页面里的 `mapXxx` 函数中调整，例如：

```typescript
// 后端返回 doctorName，前端展示用 name
name: doc.name || doc.doctorName
```
